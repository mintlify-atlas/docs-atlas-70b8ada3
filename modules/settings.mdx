The Settings module is the central control panel for every tenant in B2B Import ERP. All configuration lives in the `tenant_settings` table as typed key-value pairs, organized by module category, and protected by Row Level Security so that one tenant can never read or modify another's settings. Changes take effect immediately without requiring a deployment or server restart.

## Configuration Storage Model

Settings are persisted in the `tenant_settings` table under the `EMPRESA`, `LOCALIZACION`, `IDENTIDAD`, `DOCUMENTOS`, `ERP`, `INTEGRACIONES`, and `TELEFONIA` modules. Each row is a typed JSONB key-value pair:

```sql
-- Example rows from tenant_settings
module: 'EMPRESA',    config_key: 'nombre_comercial', config_value: '"VentiTech S.A.S."'
module: 'IDENTIDAD',  config_key: 'color_primario',   config_value: '"#0284c7"'
module: 'TELEFONIA',  config_key: 'numero_ventas',    config_value: '"+573001112233"'
```

A database trigger validates the format of each key: color values must match hex, RGB, or HSL patterns; URL values must be absolute URLs, relative paths, or base64 data URIs; phone numbers must conform to E.164 international format.

Sensitive credentials (API keys, integration secrets) are encrypted at rest using `pgp_sym_encrypt` via the `handle_tenant_settings_encryption` trigger when `is_encrypted = true`.

## Branding Versioning

Every time branding settings are saved via `saveTenantBranding()`, the system takes a full snapshot of the active branding configuration and stores it in `tenant_branding_version`. Previous versions can be restored at any point via `restoreBrandingVersion()`.

```
Version 1 → Version 2 → Version 3 (current)
                ↑
         Restore to this version → creates Version 4
```

<Note>
  Settings changes take effect immediately for server-rendered pages. Branding values cached in the browser's `localStorage` are refreshed automatically on the next full page load, so users may see the old branding for up to one navigation cycle after a change.
</Note>
